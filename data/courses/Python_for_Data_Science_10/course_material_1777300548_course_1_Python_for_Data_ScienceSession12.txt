We will cover two primary methods for acquiring data from the web: APIs (the clean, structured way) and Web Scraping (the messy, but powerful way).

1. Acquiring Data via APIs

An API (Application Programming Interface) is a formal way for two programs to talk to each other. For data scientists, APIs are the gold standard because they provide structured data (usually in JSON format).

The requests Library

The requests library is the industry standard for making HTTP calls in Python.

import requests

# Example: Fetching data from a public API
url = "[https://api.exchangerate-api.com/v4/latest/USD](https://api.exchangerate-api.com/v4/latest/USD)"
response = requests.get(url)

if response.status_code == 200:
    data = response.json()
    print(f"Base Currency: {data['base']}")
    print(f"Rate for EUR: {data['rates']['EUR']}")
else:
    print("Failed to retrieve data")


2. Web Scraping with BeautifulSoup

When an API doesn't exist, we must extract data directly from the HTML code of a website. This is called Web Scraping.

The HTML Structure

Websites are built with tags like <div>, <h1>, <a>, and <table>. We use BeautifulSoup to "parse" these tags and pull out the text or links we need.

from bs4 import BeautifulSoup
import requests

url = "[https://example.com](https://example.com)"
response = requests.get(url)
soup = BeautifulSoup(response.text, 'html.parser')

# Find the main heading
heading = soup.find('h1').text
print(f"Page Title: {heading}")

# Find all links
links = soup.find_all('a')
for link in links:
    print(link.get('href'))


3. Ethical Scraping and robots.txt

Before scraping any website, you must check its rules.

robots.txt: Visit website.com/robots.txt to see what parts of the site are off-limits to bots.

Rate Limiting: Don't hammer a server with thousands of requests per second. Use time.sleep().

Terms of Service: Always respect the legal boundaries of the data owner.
